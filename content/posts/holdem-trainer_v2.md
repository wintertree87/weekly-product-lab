> 지난번 글은 AI한테 정리시켰더니 말맛이 없더라고요. 이번엔 직접 씁니다.

홀덤 트레이너를 버전업했습니다.

처음에 HTML 파일 하나로 만든 프리플랍 퀴즈가, 이제는 로그인도 되고, 학습 기록도 쌓이고, AI 봇이랑 게임도 할 수 있는 앱이 됐어요.

기획만 잘하면 서비스가 툭툭 나오는 시대가 진짜 왔구나, 를 체감하고 있습니다.

![홀덤 트레이너 로그인 화면](/images/holdem-trainer/login.png)

## v1에서 뭐가 달라졌나

v1은 진짜 날것이었어요. HTML 단일 파일. 로그인 없고, 기록도 안 남고, 퀴즈만 풀면 끝.

v2는 꽤 다른 제품이 됐습니다.

- **듀오링고 스타일 스킬 트리** — 12개 유닛, 41개 레슨. 단계별로 잠금이 풀리고 크라운을 모으는 구조
- **승단 시험** — 자신 있으면 유닛 전체를 한 번에 통과할 수도 있어요
- **AI 봇 대전** — 봇이랑 1:1 매치. 프리플랍부터 리버까지 풀 핸드 플레이
- **XP/레벨 시스템** — 문제 풀 때마다 경험치가 쌓이고, 레벨업 하는 재미
- **카카오/구글 로그인** — 핸드폰 바꿔도 학습 기록 유지
- **확률 치트시트** — 앱 안에서 바로 확인하는 프리플랍 승률표, 드로우 확률표, 팟 오즈표

기술 스택도 갈아엎었어요. HTML 하나짜리에서 Next.js + Supabase + Vercel 조합으로. DB가 생기니까 할 수 있는 게 확 늘더라고요.

## 만드는 건 빨랐다. 유저 모으기가 진짜 어렵다

솔직히 개발은 금방금방 했어요. Claude Code 덕분에 기획만 잘하면 구현은 술술 나옵니다.

근데 진짜 어려운 건 그 다음이었어요.

**유저를 어디서 데려오지?**

포커 커뮤니티를 돌아다녀봤는데, 홍보가 쉽지 않더라고요. SEO도 건드려보고, 게시판도 올려보고, 아는 사람한테 뿌려보고. 아직 진행 중입니다.

만들어놓고 아무도 안 쓰면 의미가 없잖아요. 코딩보다 마케팅이 100배 어렵습니다.

![홀덤 트레이너 온보딩 화면](/images/holdem-trainer/onboarding.png)

## 배운 것: "만드는 구조"를 먼저 셋업해라

이번에 가장 크게 배운 거 하나.

**그냥 "만들어줘"로 시작하면 삽질한다.**

처음엔 정말 그렇게 했거든요. "이것 좀 만들어줘", "저것 좀 고쳐줘". 그러다 보니 방향이 틀어지고, 만든 걸 엎고, 비효율의 연속.

그래서 제가 원래 PO(Product Owner)로 일하면서 쓰던 프로세스를 Claude Code에 이식했습니다.

1. **전략** — 왜 만드는지, 누구한테 필요한지
2. **기획** — 뭘 만들건지, 가설과 스펙 정리
3. **디자인** — 화면 시안
4. **개발** — 코딩
5. **QA** — 테스트
6. **런칭** — 배포
7. **데이터 분석** — 가설 검증 + 피드백 수집

각 단계를 슬래시 커맨드(`/strategy`, `/plan`, `/dev`, `/review` ...)로 호출하는 시스템을 만들었더니, 생산성이 완전히 달라졌어요.

"이거 만들어줘"가 아니라 "이 단계를 수행해줘"로 바뀌니까, 결과물의 품질이 급상승합니다.

## 바이브코딩, 초입만 넘기면 급가속

Claude Code는 처음이 진짜 어렵습니다. 뭘 시켜야 하는지, 어떻게 대화해야 하는지 감이 안 와요.

근데 초입만 넘기면 러닝커브가 기하급수적으로 올라갑니다.

저는 개발자가 아닙니다. 12년차 PO예요. 코드를 직접 짜지 않아요.
근데 이제 주말에 앉으면 동작하는 제품을 하나 뚝딱 만들 수 있게 됐어요.

바이브코딩이 제 역량을 다른 수준으로 점프시켜주고 있습니다.
무서운 시대이기도, 신나는 시대이기도 해요.
